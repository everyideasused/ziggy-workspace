### Key Implementation Details

**Feedback Loop:**
- User: "I liked that" → Spin calls `logFeedback('liked_song', 'Song — Artist', 'context')` → entry written to SPIN_MEMORY.md
- User: "Skip this" → Logged to Skipped Songs section
- User: "I love [artist]" → Logged to Liked Artists
- After 3+ sessions → Spin runs taste analysis → updates Inferred Taste Summary

**Taste Learning Algorithm:**
- Session logs saved to `Notes/Spin_Sessions/YYYY-MM-DD.md`
- After 3 sessions, Spin analyzes:
  - Genre patterns in liked songs
  - BPM ranges for liked vs skipped
  - Time-of-day preferences
  - Artist patterns
  - Hard avoidances
  - Audio feature patterns (energy, valence, danceability)
- Writes 3–5 sentence plain-language summary to SPIN_MEMORY.md

**Spotify Integration:**
- Requires Spotify Premium for playback control (queue, play)
- Search, recommendations, and track info work without Premium
- Audio features (BPM, energy, valence, danceability, acousticness, key, mode) pulled via Spotify API
- Playlists auto-created in Nathan's Spotify library

**Memory Architecture:**
- SPIN_MEMORY.md: Primary memory (taste profile, session index, inferred summary)
- Spin_Sessions/YYYY-MM-DD.md: Individual session logs with tracks + feedback
- KB: Stateless (same for every session — doesn't learn, doesn't change)
- On every session start: Spin loads KB + SPIN_MEMORY.md + last 3 session logs
