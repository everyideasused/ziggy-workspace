# WF-Change-Order-Review — CO Audit Process

> Change orders are where project budgets die. A world-class estimator treats every CO with the same rigor as the original bid — because the GC knows you're under schedule pressure and is counting on you to approve quickly. Don't.

---

## The Change Order Reality Check

**80% of change orders fall into one of four categories:**

1. **Legitimate scope additions** — owner requested something new. Fair; pay it.
2. **Design errors or omissions** — the drawings were wrong or incomplete. Negotiate; typically owner pays but A/E may have liability.
3. **GC scope gaps** — the GC didn't include it in their bid but it was clearly required. Push back; this is the GC's problem.
4. **Opportunistic pricing** — real scope change, but GC priced it at 2–3× market rate knowing you can't rebid it. Negotiate hard.

**Your job:** Categorize every CO before approving any of it.

---

## Step-by-Step CO Review Process

### Step 1: Receive and Log

Every CO request goes into the log before review:

| # | Date Received | GC CO # | Description | Amount Requested | Category | Status |
|---|---|---|---|---|---|---|
| 1 | | | | $ | TBD | Under Review |

Never approve verbally. Never approve without documentation. Written CO request only.

### Step 2: Categorize the Scope

Ask three questions:
1. **Is this scope in the contract documents?** (drawings, specs, addenda)
   - If YES → It's the GC's scope. Push back.
   - If NO → Continue to step 2.
2. **Who caused the gap?** (owner change? design error? site condition?)
   - Owner change → Owner pays. Full stop.
   - Design error → Negotiate with A/E; owner may still pay but A/E has exposure.
   - Unforeseen site condition → Review contract for differing site conditions clause.
3. **Is the price reasonable?** — Audit independently regardless of cause.

### Step 3: Verify the Scope

Before touching the price, confirm the scope is accurate:
- Request backup documentation (RFI, ASI, field directive, design change)
- Confirm what specifically changed vs. original contract
- Confirm the change is not already covered by an allowance
- Confirm the change is not already covered by GC contingency (if GMP contract)

**Red flag:** GC submits CO without any backup documentation. Never approve. Require written basis.

### Step 4: Audit the Pricing

This is where most owners leave money on the table. CO pricing follows the same rules as original bids:

**Request the following from GC for any CO > $5,000:**
- [ ] Itemized labor breakdown (hours × rate × burden)
- [ ] Itemized material breakdown (quantities × unit cost)
- [ ] Sub quotes (if subcontracted work)
- [ ] Equipment costs (if applicable)
- [ ] GC markup clearly identified

**Check each component:**

**Labor:**
- Are hours reasonable for the scope? (compare to your own assessment)
- Is the labor rate reasonable? (compare to market)
- Is burden reasonable? (typically 35–50% of base wage)

**Materials:**
- Are quantities accurate? (re-measure if needed)
- Are unit costs current market prices? (not inflated)
- Any material already on-site or already paid for?

**Subcontractor quotes:**
- Is the sub quote legitimate? (not a brother-in-law estimate)
- Did GC get competitive sub pricing or single-source it?
- What markup did GC apply to the sub quote? (5–12% is typical; >15% is high)

**GC markup on CO:**
- Markup on COs is typically lower than original bid: 10–15% total
- Some contracts specify CO markup caps — check the contract
- If contract is silent: 10–15% OH&P on self-perform; 5–10% on sub work is market

### Step 5: Run Independent Pricing

For any CO > $10,000, build your own independent price:

```
INDEPENDENT CO PRICING
Change: [Description]
Date: [Date]

Labor:
  [Trade]: [X] hours × $[rate]/hr × [burden factor] = $[amount]
  
Materials:
  [Item 1]: [qty] × $[unit cost] = $[amount]
  [Item 2]: [qty] × $[unit cost] = $[amount]
  
Direct Cost: $[total]
OH&P (12%): $[amount]
─────────────────────────
Independent Total: $[total]

GC Request: $[amount]
Variance: $[amount] ([%])

Recommendation: [Approve / Negotiate to $X / Reject]
```

### Step 6: Negotiate

Never reject a CO without an offer. Always counter:
- "We've reviewed your CO and our independent analysis supports $X. We're prepared to approve $X."
- If GC pushes back: "Please provide your detailed backup so we can reconcile the difference."
- If GC can't provide backup: "We can't approve without documentation. Please resubmit with full backup."

**Negotiation leverage:**
- Schedule: if CO is on critical path, GC has leverage
- Documentation: if they can't back it up, you have leverage
- Sub quotes: if you can verify the sub's actual price, you have leverage
- Contract terms: always read CO markup caps and change order procedures in the contract

### Step 7: Approve, Partially Approve, or Reject

**Approval:** Full scope and price accepted → Execute change order, update project cost report

**Partial approval:** Scope accepted, price negotiated → Execute CO at negotiated amount

**Rejection:** Scope not a legitimate change → Document reason; GC can dispute per contract

**Time & Materials (T&M) work authorization:** For urgent undefined work, authorize T&M with a not-to-exceed cap. Require daily T&M tickets signed by superintendent and your field rep.

## CO Red Flags — Patterns That Should Trigger Scrutiny

### Pricing Red Flags

| Red Flag | What It Means |
|---|---|
| Labor rate > $95/hour all-in for standard trade | Overstated; typical fully burdened is $65–$85/hour for most commercial trades |
| Material markup > 20% | Standard is 10–15% on materials; >20% = inflated |
| GC OH&P on CO > 18% | High; market is 10–15% on COs |
| "Loaded" or "blended" rate without breakdown | GC hiding inflated component; demand itemization |
| Round numbers throughout ($10,000, $5,000, $25,000) | Lacks rigor; may be a guess not a calculation |
| Equipment rental without dates or invoices | Unverifiable; require backup |
| Sub quote on contractor letterhead only, no detail | Get the sub's actual backup |

### Scope Red Flags

| Red Flag | What It Means |
|---|---|
| CO for something clearly shown on drawings | GC scope gap; push back |
| "General conditions impact" add-on | Rarely legitimate unless schedule is formally extended |
| Multiple COs for the same general area | GC piecemealing scope to avoid scrutiny |
| CO submitted months after the work was done | Red flag — late submission loses negotiating leverage for you; get CO logs current |
| CO for work GC said was "included" during bid | Document and push back; get the bid clarification |
| Schedule impact claimed but no schedule update provided | Require updated schedule before approving any schedule impact premium |

### Timing Red Flags

| Red Flag | What It Means |
|---|---|
| CO submitted on same day as substantial completion | Desperation play; review very carefully |
| CO held until end of project (dump) | Common tactic; push for real-time CO submission in contract |
| CO for unforeseen conditions submitted long after discovery | Timely notice is typically required by contract |

## CO Approval Authority Matrix

Recommended for program management with multiple active projects:

| CO Amount | Approval Authority | Documentation Required |
|---|---|---|
| < $5,000 | PM (field) | CO form + scope description |
| $5,000–$25,000 | Program Manager (Nathan) | CO form + itemized backup + independent check |
| $25,000–$100,000 | Program Manager + Client PM | Full audit + independent estimate + client approval |
| > $100,000 | Client executive approval | Full audit + executive sign-off |

## Contract Language to Know

Before auditing any CO, know these contract provisions:
- **Markup cap on COs:** Some contracts specify maximum overhead and profit on change orders (10–15% is common)
- **Notice requirement:** Most contracts require written notice within 7–14 days of discovering a change
- **Schedule impact:** Some contracts require concurrent schedule update with any CO claiming time
- **T&M rates:** Some contracts pre-establish T&M labor rates and markup — reference these
- **Differing site conditions:** Most AIA and ConsensusDocs contracts address unforeseen conditions; know your standard
- **Dispute resolution:** Know the escalation path before disputes arise

## CO Tracking Format

Maintain a live CO log on every project:

| CO # | Date | Description | GC Request | Approved Amount | Savings | Status | Budget Line |
|---|---|---|---|---|---|---|---|
| PCO-001 | | | $X | $X | $X | Approved | MEP |
| PCO-002 | | | $X | Pending | — | Under Review | |
| PCO-003 | | | $X | $0 | $X | Rejected | |
| **TOTAL** | | | **$X** | **$X** | **$X** | | |

**Approved CO total + Original GMP = Current Contract Value**
**Current Contract Value + Owner Contingency Remaining = Project Budget Status**
