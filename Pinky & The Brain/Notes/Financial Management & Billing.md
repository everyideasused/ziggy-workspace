# Financial Management & Billing

> Cross-cutting reference for [[Construction Program Management - Master Reference]]

---

## Construction Financial Lifecycle

```
Budget Established → SOV Created → Monthly Pay Applications →
Change Orders Processed → Retainage Tracked → Final Billing →
Retainage Release → TIA Reconciliation (if applicable) → Financial Closeout
```

---

## Budget Structure

### Hard Costs (Direct Construction)
- Sitework and demolition
- Concrete and foundations
- Structural frame
- Building envelope (roofing, exterior walls, windows)
- Interior finishes (drywall, flooring, paint, ceilings)
- MEP systems (mechanical, electrical, plumbing)
- Fire protection
- Elevator/vertical transportation
- Specialties (signage, accessories, food service equipment)
- General conditions (GC's on-site costs: supervision, trailer, temp facilities, cleanup)
- GC overhead and profit (fee)
- Contingency (owner's and/or contractor's)

### Soft Costs
- Architecture and engineering fees
- Specialty consultant fees
- Permit and impact fees
- Legal fees
- Insurance premiums
- Financing costs (loan origination, interest during construction)
- FF&E (furniture, fixtures, and equipment — often separate budget)
- Technology/AV/security systems
- Moving and relocation costs
- Testing and inspection fees
- Project management / CM fees

## Schedule of Values (SOV)

The SOV is the financial backbone of every construction project. It breaks the contract into billable line items.

### SOV Structure (per AIA G703)
| Column | Content |
|--------|---------|
| Item # | Sequential line item number |
| Description of Work | Scope description (trade or activity) |
| Scheduled Value | Contract amount for this line item |
| Work Completed (Previous) | Amount billed in prior applications |
| Work Completed (This Period) | Amount billed this period |
| Materials Stored | Value of stored materials not yet installed |
| Total Completed & Stored | Cumulative amount |
| % Complete | Percentage of scheduled value completed |
| Balance to Finish | Remaining amount |
| Retainage | Amount withheld (typically 5-10%) |

### SOV Best Practices
- SOV line items should align with the scope of work and the construction schedule
- Each major trade/subcontractor should have its own line items
- General conditions should be broken out (not buried)
- GC fee/OH&P should be a separate line
- Change orders get their own line items when approved
- Total of all line items = total contract value (exactly)
- Avoid front-loading (inflating early line items to accelerate cash flow)

## Monthly Pay Application Process

### AIA G702 / G703 Workflow
```
GC updates SOV with current % complete →
Submits G702 (Application) + G703 (Continuation Sheet) →
Includes: lien waivers, stored material documentation, change orders →
Architect/CM reviews and certifies (or redlines) →
Owner approves and processes payment →
Payment issued (net of retainage)
```

### Pay Application Checklist
- [ ] G702 Application for Payment (completed and signed by GC)
- [ ] G703 Continuation Sheet (updated SOV with current billing)
- [ ] Conditional lien waiver from GC (for current draw)
- [ ] Unconditional lien waiver from GC (for previous draw)
- [ ] Conditional lien waivers from all subcontractors (current)
- [ ] Unconditional lien waivers from all subcontractors (previous)
- [ ] Stored materials documentation (if billing for stored materials)
- [ ] Approved change orders (incorporated into SOV)
- [ ] Progress photos
- [ ] Updated construction schedule (some contracts require monthly)
- [ ] Certified payroll (if prevailing wage / Davis-Bacon applies)

### Payment Timeline (Typical)
| Step | Timeline |
|------|---------|
| GC submits pay app | By 25th of month (or per contract) |
| Architect/CM review | 7-10 days |
| Owner approval | 5-7 days |
| Payment issued | 30 days from receipt (or per contract) |
| **Total cycle** | **30-45 days from submission** |

## Change Order Management

### Change Order Lifecycle
```
Issue Identified → PCO (Potential Change Order) Logged →
Scope Defined (ASI, RFI response, owner directive) →
GC Solicits Pricing from Subs → GC Compiles COR →
CM/Architect Reviews Pricing → Negotiation (if needed) →
Owner Approves → Formal Change Order Executed →
SOV Updated → Billed in Next Pay Application
```

### Change Order Pricing Components
- Direct labor (hours × rate, including burden)
- Materials (at cost, plus markup per contract)
- Equipment (owned rates or rental rates)
- Subcontractor costs (sub's quote + GC markup per contract)
- GC overhead and profit (typically 10-15% on self-performed; 5-10% on sub work)
- Architect/engineer additional services (if redesign required)
- Schedule impact (time extension request, if applicable)

### Change Order Documentation
- Written description of changed scope
- Reference to RFI, ASI, or owner directive that triggered the change
- Detailed cost breakdown with backup
- Schedule impact analysis
- Before/after comparison (if applicable)

## Retainage

### Purpose
Provides owner leverage to ensure contractor completes all work, including punch list and closeout obligations.

### Typical Structure
- 10% retained from each pay application during construction
- Reduced to 5% at substantial completion (common but must be contractual)
- Final retainage released upon: punch list completion, CO, final lien waivers, closeout documents

### State Law Considerations
- Many states have retainage limits (commonly 5% max on public projects)
- Some states require retainage to be held in escrow or interest-bearing account
- Release timelines may be legislated
- **Always check state-specific retainage laws for the project jurisdiction**

## TIA Financial Management

See [[Tenant Improvement Allowances]] for full coverage. Key financial touchpoints:

- TIA amount established in lease
- Disbursement method defined (progress payments vs. lump sum at completion)
- Documentation requirements for each draw
- Reconciliation at project completion
- Overage responsibility (tenant pays above TIA cap)
- Unused balance disposition (credit to rent or reverts to landlord)

## Cost-to-Complete Forecasting

Monthly exercise to project the final cost of the project:

```
Original Contract + Approved COs + Pending COs + Anticipated COs + 
Remaining Contingency Draw = Projected Final Cost
```

### Forecast Components
| Component | Source |
|-----------|--------|
| Original contract value | Executed contract |
| Approved change orders | CO log |
| Pending change orders | PCO log (estimated) |
| Anticipated changes | PM judgment based on known issues |
| Owner contingency remaining | Budget tracking |
| Contractor contingency remaining | GC reporting (GMP projects) |
