# AHJ Research Methodology

> Cross-cutting reference for [[Construction Program Management - Master Reference]] and [[04 - Entitlements & Permitting]]

---

## What is an AHJ?

An **Authority Having Jurisdiction (AHJ)** is any organization, office, or individual responsible for enforcing codes, standards, and regulations within a defined geographic or functional area. The term comes from NFPA 101 and is used broadly across all building codes.

---

## Framework: Researching Any New Jurisdiction

When encountering a new city, county, or municipality for the first time, follow this systematic approach:

### Step 1: Identify the Governing AHJs

| AHJ Type | What They Control | How to Find Them |
|----------|------------------|------------------|
| **Building Department** | Building permits, structural/MEP/architectural plan review, inspections, CO | City or county government website → "Building" or "Development Services" |
| **Fire Marshal / Fire Department** | Fire and life safety review, fire alarm/sprinkler permits, fire inspections | Often part of fire department; sometimes within building department |
| **Planning / Zoning Department** | Land use, zoning compliance, variances, CUPs, site plan review | City or county government website → "Planning" or "Community Development" |
| **Health Department** | Food service, pools, medical facilities, childcare | County or state health department |
| **Department of Transportation (DOT)** | Driveway access, traffic studies, ROW permits | State DOT or city/county public works |
| **Environmental / Stormwater** | Stormwater management, erosion control, wetlands | City engineering, county conservation district, state environmental agency |
| **Utility Companies** | Service connections, capacity, meter installation | Local electric, gas, water/sewer, telecom providers |
| **Historic Preservation** | Projects in historic districts or involving historic structures | City historic preservation commission or state SHPO |
| **State Agencies** | State fire marshal (some states), elevator inspections, accessibility, state building code | Varies by state |

### Step 2: Determine Code Editions in Effect

| Code Type | Common Editions | Who Adopts |
|-----------|----------------|------------|
| Building Code | IBC (International Building Code) — 2018, 2021, 2024 | State or local adoption |
| Fire Code | IFC (International Fire Code) | State or local adoption |
| Mechanical Code | IMC (International Mechanical Code) | State or local adoption |
| Plumbing Code | IPC or UPC (varies by region) | State or local adoption |
| Electrical Code | NFPA 70 (NEC — National Electrical Code) — 2020, 2023 | State or local adoption |
| Energy Code | IECC (International Energy Conservation Code) or ASHRAE 90.1 | State or local adoption |
| Accessibility | ADA (federal); state accessibility codes may exceed ADA | Federal + state |
| Fire Protection | NFPA 13 (sprinklers), NFPA 72 (fire alarm) | Via IBC/IFC adoption |

**Critical:** Many jurisdictions adopt codes with local amendments. Always verify the specific edition AND amendments in effect.

### Step 3: Gather Permit Process Information

For each jurisdiction, document:

- [ ] **Permit types required** (building, mechanical, electrical, plumbing, fire, demo, sign, grading)
- [ ] **Application requirements** (forms, number of drawing sets, digital submission options)
- [ ] **Plan review timeline** (standard and expedited options)
- [ ] **Plan review fees** (typically based on construction valuation)
- [ ] **Impact fees** (transportation, parks, schools, water/sewer)
- [ ] **Contractor licensing requirements** (state license, local registration)
- [ ] **Inspection process** (who schedules, how far in advance, who attends)
- [ ] **Pre-application meeting availability** (strongly recommended for new jurisdictions)
- [ ] **Third-party review acceptance** (some jurisdictions accept private plan review firms)
- [ ] **Phased permitting availability** (foundation permit before full building permit)
- [ ] **Online portal** (many jurisdictions now use Accela, Tyler, or similar for e-permitting)

### Step 4: Document Jurisdiction-Specific Quirks

Every jurisdiction has unique requirements. Examples:
- **Fire sprinkler thresholds:** Some require sprinklers at 5,000 SF; IBC threshold may differ
- **Parking ratios:** Vary significantly (e.g., 1:250 SF for retail vs. 1:1,000 SF for office)
- **Sign regulations:** Height, area, illumination, setback — all locally determined
- **Design review boards:** Some jurisdictions require architectural review for aesthetics
- **Noise ordinances:** Construction hours, equipment restrictions
- **Tree preservation:** Replacement requirements, protected species
- **Water conservation:** Low-flow fixture requirements, irrigation restrictions

## AHJ Relationship Management Best Practices

1. **Pre-application meetings** are almost always available and almost always worth it
2. **Respect the reviewer's time** — submit complete, organized packages
3. **Respond to comments thoroughly** — address every comment, even if you disagree (explain why)
4. **Don't surprise the building department** — if something unusual is planned, discuss it early
5. **Be professional and patient** — building departments are often understaffed
6. **Track your reviewer** — knowing who reviews your plans allows you to build rapport
7. **Understand the chain of command** — if a reviewer's interpretation seems wrong, the chief building official can weigh in
8. **Document verbal guidance** — follow up phone calls/meetings with written confirmation

## Multi-Jurisdiction Program Management

When managing a program across many jurisdictions (e.g., national retail rollout):

- Maintain a **jurisdiction database** with all Step 1-5 information per location
- Develop **prototype drawing sets** with jurisdiction-specific amendment pages
- Identify common jurisdictional variations and pre-engineer solutions
- Budget for permit timeline variability (some cities = 2 weeks; some = 6+ months)
- Consider **permit expediting services** in notoriously slow jurisdictions
- Track permit status across all projects in a centralized dashboard
